
## Declaring Variables in Python

In Python, a **variable** is a name that refers to a value stored in memory.
You **declare a variable by assigning a value** to it using the assignment operator (`=`).

### Syntax

```python
variable_name = value
```

### Example

```python
name = "John Doe"
age = 25
```

* `name` stores a **string**
* `age` stores an **integer**
* Python automatically understands the data type, so no extra keywords are required

---

## Rules for Naming Variables in Python

Python has strict rules for variable names:

1. **Must start with a letter or underscore**

   ```python
   _count = 10
   name = "Alex"
   ```

   ❌ Not allowed:

   ```python
   5number = 10
   ```

2. **Can contain only letters, numbers, and underscores**

   ```python
   user_age = 30
   ```

3. **Case-sensitive**

   ```python
   age = 20
   Age = 25
   ```

   (`age` and `Age` are different variables)

4. **Cannot use Python keywords**
   ❌ Examples:

   ```python
   if = 10
   class = "Test"
   ```

If rules are broken, Python raises a `SyntaxError`.

---

## Variable Naming Conventions (Best Practices)

### 1. Use **snake_case**

* Lowercase letters
* Words separated by underscores

```python
my_variable_name = "freeCodeCamp"
```

---

### 2. Use **descriptive names**

Bad:

```python
x = 30
```

Good:

```python
user_age = 30
```

Descriptive names make code easier to understand and maintain.

---

### 3. Avoid single-letter names

❌ Avoid:

```python
x = 56
```

✅ Acceptable only in loops:

```python
for i in range(5):
    print(i)
```

---

## Comments in Python

Comments are used to explain code. Python **ignores comments during execution**.

### Single-line comment

```python
# This is a comment
```

### Multi-line comment

```python
# This is a
# multi-line
# comment
```

### Example

```python
user_age = 30  # stores the age of the user
```

---

## Important Tip

❌ Don’t use comments to explain **poor variable names**
✅ Use **clear and meaningful variable names instead**

---

* Variables are created using `=`

* Use `snake_case` and descriptive names
