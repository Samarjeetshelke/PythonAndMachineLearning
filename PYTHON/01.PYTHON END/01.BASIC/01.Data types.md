## 1️⃣ What are Data Types in Python?

A **data type** tells Python **what kind of value** a variable is holding so Python knows:

* how to store it in memory
* what operations are allowed on it

### Example:

```python
age = 25
```

Here, Python understands:

* `25` is a **number**
* specifically an **integer**
* so it allows math operations like `+`, `-`, `*`, `/`

---

## 2️⃣ Python Is a Dynamically Typed Language

### What does *dynamically typed* mean?

In Python:

* You **do NOT** tell Python the data type explicitly
* Python **automatically decides the data type** based on the value you assign

### Example:

```python
name = "John Doe"
age = 25
```

Python internally understands:

* `"John Doe"` → `str` (string)
* `25` → `int` (integer)

You didn’t write:

```python
string name
int age
```

Python figured it out on its own.

---

### Contrast with Statically Typed Languages

Languages like **C, C++, Java, C#** are *statically typed*.

There you MUST specify the type:

```java
int age = 25;
String name = "John";
```

---

## 3️⃣ Advantage & Disadvantage of Dynamic Typing

### ✅ Advantages

* Faster coding
* Cleaner and shorter code
* More flexible

### ❌ Disadvantages

* Errors appear **at runtime**, not before
* Some bugs are found **only when the program reaches that line**

### Example of a runtime error:

```python
x = 10
x = "hello"
print(x + 5)  # ❌ Error at runtime
```

Python crashes only **when this line runs**, not before.

## 4️⃣ Common Data Types in Python (Explained Clearly)

---

### 🔹 Integer (`int`)

Whole numbers without decimals.

```python
x = 10
y = -5
```

Used for:

* counting
* indexing
* loop counters

---

### 🔹 Float (`float`)

Numbers with decimals.

```python
price = 99.99
temperature = -4.5
```

Used for:

* measurements
* calculations requiring precision

---

### 🔹 String (`str`)

Text data enclosed in quotes.

```python
name = "Alice"
message = 'Hello World'
```

Used for:

* names
* messages
* file paths

---

### 🔹 Boolean (`bool`)

Only two values:

* `True`
* `False`

```python
is_logged_in = True
```

Used for:

* conditions
* decision making

---

### 🔹 List (`list`)

Ordered, mutable collection (can change).

```python
items = [1, "apple", 3.5, True]
```

Features:

* can store mixed data types
* can be modified

---

### 🔹 Tuple (`tuple`)

Ordered, **immutable** collection.

```python
coordinates = (10, 20)
```

Cannot be changed once created.

---

### 🔹 Set (`set`)

Unordered collection of **unique** values.

```python
numbers = {1, 2, 3, 3}
# Output: {1, 2, 3}
```

Duplicates are removed automatically.

---

### 🔹 Dictionary (`dict`)

Key–value pairs.

```python
user = {
    "name": "Alice",
    "age": 25
}
```

Used to store structured data.

---

### 🔹 Range (`range`)

Sequence of numbers, mainly used in loops.

```python
range(5)  # 0, 1, 2, 3, 4
```

---

### 🔹 None (`NoneType`)

Represents **no value**.

```python
result = None
```

Used when:

* a variable exists but has no value yet
* a function returns nothing

---

## 5️⃣ Checking the Data Type → `type()`

You can check a variable’s data type using `type()`.

```python
x = 10
print(type(x))
```

Output:

```text
<class 'int'>
```

More examples:

```python
type("hello")   # str
type(3.14)      # float
type(True)      # bool
```

---

## 6️⃣ Checking Type Safely → `isinstance()`

`isinstance()` checks **if a variable is of a specific type** and returns `True` or `False`.

```python
isinstance(10, int)       # True
isinstance("hi", str)     # True
isinstance(10, float)    # False
```

✅ Preferred in real programs because it’s safer and more flexible.
